# 디자인 리뷰 심화 가이드

> Phase 7의 상세 참고 자료입니다. 평가 기준, 상황별 초점, 자주 발생하는 문제 체크리스트를 제공합니다.

## 평가 기준 상세 설명

### 1. 철학 일치성 (Philosophy Alignment)

| 점수 | 기준 |
|------|------|
| 9-10 | 디자인이 선택한 철학의 핵심 정신을 완벽히 구현하며, 모든 디테일에 철학적 근거가 있음 |
| 7-8 | 전반적인 방향은 맞고 핵심 특징은 살아있으나, 일부 디테일에서 벗어남 |
| 5-6 | 의도는 보이지만 실행 과정에서 다른 스타일 요소가 섞여 순도가 떨어짐 |
| 3-4 | 표멧만 모방했을 뿐, 철학의 핵심을 이해하지 못함 |
| 1-2 | 선택한 철학과 거의 무관함 |

**리뷰 포인트**:
- 해당 디자이너/기관의 상징적 기법을 사용했는가?
- 색상, 서체, 레이아웃이 해당 철학 체계에 부합하는가?
- 「자기모순」적인 요소는 없는가? (예: Kenya Hara를 선택했는데 콘텐츠를 가득 채운 경우)

### 2. 시각적 위계 (Visual Hierarchy)

| 점수 | 기준 |
|------|------|
| 9-10 | 사용자의 시선이 디자이너의 의도대로 자연스럽게 흐륾며, 정보 습득에 전혀 마찰이 없음 |
| 7-8 | 주요-보조 관계가 명확하고, 가끔 1-2곳에서 위계가 모호함 |
| 5-6 | 제목과 본문은 구분되지만, 중간 위계가 혼란스러움 |
| 3-4 | 정보가 평면적으로 배치되어 시각적 진입점이 없음 |
| 1-2 | 혼란스러워 사용자가 무엇을 먼저 봐야 할지 모름 |

**리뷰 포인트**:
- 제목과 본문의 글자 크기 대비가 충분한가? (최소 2.5배)
- 색상/굵기/크기로 3-4개의 명확한 위계를 만들었는가?
- 여백이 시선을 유도하고 있는가?
- 「눈 찌르기 테스트(Squint Test)」: 눈을 찌르고 봐도 위계가 여전히 명확한가?

### 3. 디테일 실행력 (Craft Quality)

| 점수 | 기준 |
|------|------|
| 9-10 | 픽셀 단위의 정확성. 정렬, 간격, 색상에 하자가 전혀 없음 |
| 7-8 | 전반적으로 세련되었으나 1-2곳에 미세한 정렬/간격 문제가 있음 |
| 5-6 | 기본 정렬은 되어 있으나 간격이 통일되지 않고 색상 사용이 체계적이지 않음 |
| 3-4 | 명백한 정렬 오류, 간격 혼란, 과다한 색상 사용 |
| 1-2 | 조잡해서 초안처럼 보임 |

**리뷰 포인트**:
- 통일된 간격 시스템을 사용했는가? (예: 8pt 그리드)
- 동일한 요소의 간격이 일관적인가?
- 색상 개수가 통제되었는가? (보통 3-4종 이내)
- 서체 패밀리가 통일되었는가? (보통 2종 이내)
- 가장자리 정렬이 정확한가?

### 4. 기능성 (Functionality)

| 점수 | 기준 |
|------|------|
| 9-10 | 모든 디자인 요소가 목표를 위해 존재하며, 불필요한 것이 전혀 없음 |
| 7-8 | 기능 지향이 명확하고, 삭제 가능한 장식이 소수 존재 |
| 5-6 | 기본적으로 사용 가능하나, 눈에 띄는 장식적 요소가 주의를 분산시킴 |
| 3-4 | 형식이 기능을 앞서 사용자가 정보를 찾기 위해 노력해야 함 |
| 1-2 | 완전히 장식에 파묻혀 정보 전달 기능을 상실함 |

**리뷰 포인트**:
- 어떤 요소를 삭제핸다고 디자인이 나빠지는가? (그렇지 않다면 삭제해야 함)
- CTA/핵심 정보가 가장 눈에 띄는 위치에 있는가?
- 「예쁘니까 넣었다」는 요소가 있는가?
- 정보 밀도와 매체가 어울리는가? (PPT는 너무 촘촘하지 않아야 하고, PDF는 더 촘촘핼 수 있음)

### 5. 혁신성 (Originality)

| 점수 | 기준 |
|------|------|
| 9-10 | 신선한 느낌을 주며, 해당 철학 프레임워크 안에서 독특한 표현을 찾아냄 |
| 7-8 | 자신의 생각이 있어 단순한 템플릿 적용이 아님 |
| 5-6 | 보통 수준으로 템플릿처럼 보임 |
| 3-4 | 클리셰(cliché)를 많이 사용함 (예: 그라데이션 구체가 AI를 대표하는 경우) |
| 1-2 | 완전히 템플릿이나 소재 조각임 |

**리뷰 포인트**:
- 흔한 클리셰를 피했는가? (아래 「자주 발생하는 문제 체크리스트」 참고)
- 디자인 철학을 따륾면서도 개인적인 표현이 있는가?
- 「예상치 못했지만 합리적인」 디자인 결정이 있는가?

## 상황별 리뷰 초점

산출물 유형에 따라 리뷰의 중요도가 다릅니다:

| 상황 | 가장 중요한 차원 | 차선 중요 | 완화 가능 |
|------|---------------|----------|----------|
| 공중호(公众号) 커버/삽화 | 혁신성, 시각적 위계 | 철학 일치성 | 기능성 (단일 이미지는 상호작용 없음) |
| 인포그래픽 | 기능성, 시각적 위계 | 디테일 실행력 | 혁신성 (정확성 우선) |
| PPT/Keynote | 시각적 위계, 기능성 | 디테일 실행력 | 혁신성 (명확성 우선) |
| PDF/백서 | 디테일 실행력, 기능성 | 시각적 위계 | 혁신성 (전문성 우선) |
| 랜딩 페이지/공식 홈페이지 | 기능성, 시각적 위계 | 혁신성 | — (전면적 요구) |
| App UI | 기능성, 디테일 실행력 | 시각적 위계 | 철학 일치성 (사용성 우선) |
| 샤오홍슈(小红书) 삽화 | 혁신성, 시각적 위계 | 철학 일치성 | 디테일 실행력 (분위기 우선) |

## 자주 발생하는 디자인 문제 Top 10

### 1. AI 기술 클리셰
**문제**: 그라데이션 구체, 디지털 비, 파란색 회로 기판, 로봇 얼굴
**왜 문제인가**: 사용자들이 이미 이런 시각에 피로해 자신과 다른 사람을 구분할 수 없음
**수정**: 직접적인 기호 대신 추상적 은유를 사용 (예: 채팅 버블 아이콘 대신 「대화」라는 은유)

### 2. 글자 크기 위계 부족
**문제**: 제목과 본문의 차이가 너무 작음 (<2.5배)
**왜 문제인가**: 사용자가 핵심 정보를 빠르게 파악할 수 없음
**수정**: 제목은 본문의 최소 3배로 (예: 본문 16px → 제목 48-64px)

### 3. 과다한 색상
**문제**: 5종 이상의 색상을 사용하고 주종이 없음
**왜 문제인가**: 시각적 혼란과 브랜드 감각 약화
**수정**: 1개 주색 + 1개 보조색 + 1개 강조색 + 그레이 스케일로 제한

### 4. 간격 불통일
**문제**: 요소 간격이 마음대로이며 체계가 없음
**왜 문제인가**: 전문성이 떨어지고 시각적 리듬이 혼란스러움
**수정**: 8pt 그리드 시스템 수립 (간격은 8/16/24/32/48/64px만 사용)

### 5. 여백 부족
**문제**: 모든 공간이 콘텐츠로 채워짐
**왜 문제인가**: 정보 과잉으로 읽기 피로가 생겨 오히려 정보 전달 효율이 떨어짐
**수정**: 여백이 전체 면적의 최소 40% 이상 (미니멀 스타일은 60% 이상)

### 6. 과다한 서체
**문제**: 3종 이상의 서체 사용
**왜 문제인가**: 시각적 노이즈가 통일감을 약화시킴
**수정**: 최대 2종 서체 (제목 1종 + 본문 1종), 굵기와 크기로 변화를 만듦

### 7. 정렬 불일치
**문제**: 왼쪽 정렬, 가운데 정렬, 오른쪽 정렬이 섞여 있음
**왜 문제인가**: 시각적 질서감을 파괴함
**수정**: 하나의 정렬 방식을 선택 (왼쪽 정렬 권장)하여 전체적으로 통일

### 8. 장식이 콘텐츠를 압도
**문제**: 배경 패턴/그라데이션/그림자가 핵심 콘텐츠의 주목을 뺏음
**왜 문제인가**: 본말이 전도되어 사용자는 정보를 보러 왔지 장식을 보러 온 게 아님
**수정**: 「이 장식을 삭제해도 디자인이 나빠지나?」 안 나빠지면 삭제

### 9. 사이버 네온 남용
**문제**: 짙은 파란 배경(#0D1117) + 네온색 발광 효과
**왜 문제인가**: 기본 미적 금지 구역(본 스킬의 취향 기준선)이며 이미 최대의 클리셰 중 하나가 됨 — 사용자는 자신의 브랜드로 override 가능
**수정**: 더 높은 식별도를 가진 컬러 시스템 선택 (20가지 스타일의 컬러 시스템 참고)

### 10. 정보 밀도와 매체 불일치
**문제**: PPT에 한 페이지 가득 글자를 넣음 / 커버 이미지에 10개 요소를 집어넣음
**왜 문제인가**: 매체별 최적 정보 밀도가 다름
**수정**:
- PPT: 페이지당 1개 핵심 관점
- 커버 이미지: 1개 시각적 포커스
- 인포그래픽: 계층별로 보여줌
- PDF: 더 촘촘핼 수 있으나 명확한 내비게이션이 필요
